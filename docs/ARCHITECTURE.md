## Design Decisions

### Why SQS FIFO over Kinesis ($11/mo saved)
Kinesis is for high-throughput streaming (>1K msgs/sec). We process a few hundred
articles every 15 minutes. SQS FIFO gives us exactly-once delivery and
content-based deduplication for free. At our volume, cost is effectively $0.

### Why Aurora Serverless v2 over regular RDS
Scales to near-zero when idle. A demo costs cents instead of the $15/mo minimum
for a db.t4g.micro running 24/7. The 0.5 ACU minimum is $0.12/hr only when
active--with auto-pause, overnight cost approaches zero.

### Why not DynamoDB (v1 used it)
DynamoDB is great for key-value access patterns. Intent scoring needs:
- Time-range queries on signals (last 90 days per company)
- Aggregations across signal types
- Score history as a time-series
- JOINs for the API layer (company + signals + contacts)

PostgreSQL handles all of these natively. DynamoDB would need GSIs, scan
operations, and application-side joins.

### Why separate Scorer Lambda (not inline in Extractor)
Separation of concerns: extractor deals with unstructured→structured,
scorer deals with business logic. If the scoring model changes (new weights,
new signal types), only the scorer deploys. Also enables batch re-scoring
without re-extracting.

### Alert cooldown
Borrowed from financial signal processing: when a company crosses the hot-lead
threshold, suppress further alerts for 6 hours. New signals still update the
score--they just don't re-fire notifications. Prevents alert fatigue when a
funding announcement + 3 job postings hit the same day.

## Cost Model

| Component | Idle | Active (demo) | At scale (10K companies) |
|-----------|------|---------------|--------------------------|
| Aurora Serverless | $0 (paused) | ~$0.12/hr | ~$50/mo |
| Lambda (4 functions) | $0 | ~$0.50/demo | ~$5/mo |
| SQS FIFO | $0 | ~$0.01 | ~$1/mo |
| EventBridge | $0 | $0 | $0 |
| SNS | $0 | ~$0.01 | ~$0.50/mo |
| Fargate (API) | $0 (stopped) | ~$0.04/hr | ~$15/mo |
| **Total** | **$0** | **$2-5/session** | **~$70/mo** |

vs. production equivalent: $15K/mo (Salesforce + ZoomInfo + 6sense + custom dev)

## Security

- All Lambda functions run in VPC private subnets
- RDS not publicly accessible--VPC-internal only
- IAM follows least-privilege: each Lambda gets only the specific queue/topic/model it needs
- Database credentials generated by Terraform, stored in state (use Secrets Manager for prod)
- No hardcoded credentials anywhere in application code
